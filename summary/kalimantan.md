---

## PROMPT LANDING PAGE (INSPIRED BY WONDERFUL INDONESIA – KALIMANTAN)

> **Role**
> Kamu adalah **Senior UI/UX Designer + Frontend Engineer (Next.js/React)** yang berpengalaman membuat landing page pariwisata berskala nasional.
>
> ---
>
> **Objective**
> Buat sebuah **landing page pariwisata** yang **terinspirasi dari tampilan dan struktur halaman**:
> `wonderfulindonesia.co.id/category/kalimantan/`
>
> Landing page ini bertujuan untuk **menampilkan destinasi wisata daerah** dengan kesan:
>
> * Alam
> * Budaya
> * Eksploratif
> * Premium & modern
>
> ---
>
> ### 1. Visual Style & UI
>
> * Full-width layout
> * Hero section dengan **background image besar / slideshow**
> * Overlay text dengan judul besar & deskripsi singkat
> * Warna natural (hijau, biru, earth tone)
> * Typography modern & clean
> * Smooth scroll & subtle animation (fade-in, slide-up)
> * Card-based layout untuk konten
>
> ---
>
> ### 2. Struktur Halaman
>
> #### A. Hero Section
>
> * Background image/video bertema alam
> * Title besar (nama wilayah)
> * Subtitle deskripsi singkat
> * Breadcrumb / category label
>
> #### B. Section Konten Utama
>
> Tampilkan **grid artikel/destinasi** dengan:
>
> * Thumbnail image
> * Category tag
> * Judul artikel
> * Deskripsi singkat (2–3 baris)
> * Button “Read More”
>
> Layout:
>
> * Desktop: 3 kolom
> * Tablet: 2 kolom
> * Mobile: 1 kolom
>
> ---
>
> #### C. Featured Section (Opsional)
>
> * Highlight 1–2 destinasi unggulan
> * Layout horizontal (image kiri, teks kanan)
>
> ---
>
> #### D. Pagination / Load More
>
> * Tombol “Load More” atau pagination numerik
>
> ---
>
> ### 3. Interaksi & UX
>
> * Hover effect pada card
> * Image lazy loading
> * Smooth transition saat load more
> * Clickable card (entire card bisa diklik)
>
> ---
>
> ### 4. Teknologi & Implementasi
>
> * **Framework**: React / Next.js
> * **Styling**: Tailwind CSS atau SCSS
> * **Component-based architecture**
>
> Contoh komponen:
>
> * `HeroSection`
> * `DestinationCard`
> * `DestinationGrid`
> * `FeaturedDestination`
> * `Pagination`
>
> ---
>
> ### 5. Konten
>
> * Gunakan **dummy data JSON**
> * Struktur data:
>
> ```json
> {
>   "title": "",
>   "category": "",
>   "image": "",
>   "excerpt": "",
>   "slug": ""
> }
> ```
>
> ---
>
> ### 6. Best Practice
>
> * SEO friendly (H1, meta title, meta description)
> * Responsive & mobile-first
> * Aksesibilitas dasar (alt image, contrast)
> * Clean & reusable code
>
> ---
>
> **Output yang Diharapkan**
>
> * Source code landing page
> * Struktur komponen
> * Dummy data
> * Preview layout (deskripsi visual)
>
> **Catatan**
>
> * Jangan menyalin konten asli
> * Fokus pada **gaya visual & struktur**
> * Asumsikan ini untuk **pembelajaran & portofolio**
>
> **Mulai dari:**
>
> 1. Breakdown UI section
> 2. Component hierarchy
> 3. Implementasi landing page
